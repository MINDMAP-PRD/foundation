# BUILD-PLAN ARCHIVE - NON-FOUNDATION

This folder contains the `BUILD-PLAN` files that do not belong in the shared base foundations.

They were moved out of `BUILD-PLAN/` so that `BUILD-PLAN/` stays focused on comprehensive `GeneralApp` foundation scope.

## Why these files were moved

These archived files are still useful, but they are one of:

- module-specific systems
- optional future shared capabilities rather than base foundation scope
- meta/reference corpora
- training/profiling material rather than canonical runtime contracts

## Examples

- `KANBAN_IMPLEMENTATION.md`
- `comprehensive-mindmap-flow.md`
- `SCOUTS_IMPLEMENTATION.md`
- project-profiler and industry-profile documents

## Rule

Archive status does not mean "bad" or "unused."

It means:

- do not treat these as mandatory base-foundation build scope
- only promote them later through explicit foundation decisions
- keep the active `BUILD-PLAN/` folder focused on shared platform capabilities
