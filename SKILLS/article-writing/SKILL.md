# Article Writing

Write long-form content that sounds like a real person or brand, not generic AI output.

This skill is for writing quality and voice, not for document file generation or PDF layout.

Use it alongside:

- `foundation-module-factory` when a governed package includes guides, launch docs, tutorials, or adoption content
- `doc` when the final deliverable should be a `.docx`
- `pdf` when the final deliverable should be a PDF

## Read First

Before writing repo-grounded long-form content, read:

1. [`../SKILL_INTERCONNECTION_MAP.md`](../SKILL_INTERCONNECTION_MAP.md)
2. the governed package, if one already exists
3. the source materials the user provided
4. [`references/long-form-writing.md`](references/long-form-writing.md)

If the article is about a foundation-based module or project in this repo, read the governed package first and stay grounded in that package and its source files.

## When To Activate

Activate this skill when the task involves:

- drafting blog posts, essays, guides, tutorials, launch posts, or newsletter issues
- turning notes, transcripts, or rough research into polished long-form content
- matching an existing founder, operator, or brand voice from examples
- tightening structure, pacing, evidence, and credibility in already-written long-form copy

## Core Rules

1. Lead with something concrete: example, output, anecdote, number, screenshot description, or code block.
2. Explain after the example, not before.
3. Prefer short, direct sentences over padded ones.
4. Use specific sourced numbers when available.
5. Never invent biographical facts, company metrics, user stories, or evidence.
6. If the writing is repo-grounded, tie claims back to the provided package, docs, or source material.

## Voice Capture Rule

If the user wants a specific voice, gather one or more of:

- published articles
- newsletters
- posts
- docs or memos
- a short style guide

Extract:

- sentence length and rhythm
- formality level
- favored rhetorical devices
- tolerance for humor, opinion, and contrarian framing
- formatting habits such as headers, bullets, code blocks, and pull quotes

If no voice references are given, default to a direct, operator-style voice: concrete, practical, and low on hype.

## Output Expectations

When this skill is used, it should leave behind one or more of the following:

- a polished long-form draft
- a structural rewrite of an existing article
- a voice-matched rewrite grounded in source material
- a sectioned outline that is ready to expand into a full article
